## 🌱チームでの分野別学習戦略

#### テクノロジ系
- **学習方法**: 参考書を読みながら学んだことを**喋りながらアウトプット**
- **記録**: 理解した内容や疑問点をIssueに記録
- **実践**: ある程度理解したら**過去問を一緒に解く**
- **担当制**: 各自が担当分野を決めて**お互いに解説し合う**

#### セキュリティ・ネットワーク
- **インプット**: **単語帳**で基礎用語を徹底暗記
- **アウトプット**: 単語帳と過去問を往復
- **補強**: 動画や参考書でのインプットも併用

#### ストラテジ系
- **重点**: **参考書と単語帳でインプット**に勤める
- **実践**: 単語帳と過去問を往復
- **補強**: 動画や参考書でのインプットも併用

#### マネジメント系
- **基礎**: 参考書を読んで概念理解
- **実践**: **ひたすら過去問で問題に慣れる**

---

<br><br><br>

#### 毎朝の習慣
- **過去問共有**: 毎朝**過去問を1問以上共有し合う**
- その日のうちに問題や分野についての解答・解説・疑問点を何かしらの形で議論

## 🌱Issue管理システム

**以下のラベルと3つのIssueテンプレートが組み込まれています。**  
活用例を参考にIssue内で学習管理と振り返りをしてください。
<br><br>

### ◉使い方
1. **"New Issue"** をクリック
2. **3つのテンプレート**から選択
3. **ラベルが自動設定**される
4. **テンプレートに沿って記入**
<br><br>

### ◉ラベル一覧

##### 大分類ラベル
- `Planning` - 学習計画
- `exercise` - 問題演習  
- `Knowledge` - 学んだことノート

##### 分野・内容ラベル
- `アルゴリズム` - アルゴリズム問題
- `重点問題` - 重要問題・つまづいた分野
- `学んだこと` - 解説・共有
- `単語` - 重要用語
- `気づき` - 発見・理解

##### 進捗・レベルラベル
- `難問` - 理解に時間がかかる問題
- `要復習` - 間違えた問題の再挑戦
- `解決済み` - 理解完了した問題
<br><br>
### ◉Issue分類体系

#### 💫学習計画 - Planning
期間ごとの計画等を記載、ラベル等で自由にカスタマイズする

#### 💫問題演習 - exercise
**学習方法は3択:**
1. **アプリ** - 過去問道場等のアプリで個人学習
2. **参考書** - 参考書の問題を個人で解く
3. **Issue協働** - GitHubで共同編集者と一緒に解く（重要問題・アルゴリズム）

- 分野（タイトル表示）
    - `アルゴリズム` - アルゴリズム問題の協働解答・解説
    - `重点問題` - FE試験で特に重要な問題、つまづいた分野の問題
- 進捗・レベルラベル
    - `難問` - 理解に時間がかかる問題
    - `要復習` - 間違えた問題の再挑戦
    - `解決済み` - 理解完了した問題

#### 💫学んだことノート - Knowledge
- 内容（タイトル表示）
    - `学んだこと` - 担当分野の解説・共有
    - `単語` - 特に覚えておきたい用語
    - `気づき` - 学習中の重要な発見や理解
<br><br>
### ◉Issue活用例

#### 学習計画例
```
タイトル: [週間計画] 2024/10/7-13 学習計画
ラベル: Planning

内容:
## 今週の目標
- [ ] テクノロジ: データベース正規化（@rin5uron担当）
- [ ] 毎朝過去問: 5問以上共有
- [ ] アルゴリズム: ソート系3問協働解答

## 分野別計画
- **テクノロジ**: 参考書Ch.5 + 解説Issue作成
- **セキュリティ**: 単語帳50語 + 過去問10問
- **アルゴリズム**: バブルソート完全理解
```

#### 問題演習例

##### アルゴリズム問題
```
タイトル: [アルゴリズム] バブルソート完全理解
ラベル: アルゴリズム, exercise

内容:
## 問題
配列 A = [5,2,8,1] をバブルソートで昇順に並び替える過程を示せ

## 解答・考察
- @rin5uron: 1回目のループで...
- @共同編集者: ポイントは隣接要素の比較で...

## 理解のポイント
- 時間計算量: O(n²)
- 安定ソート
- 実装が簡単だが効率は悪い
```

##### 重点問題例
```
タイトル: [重点問題] データベース正規化
ラベル: 重点問題, exercise

内容:
## 問題（FE過去問より）
以下の非正規化テーブルを第3正規形まで正規化せよ...

## 解答プロセス
1. 第1正規形: 繰り返し項目の除去
2. 第2正規形: 部分関数従属の除去
3. 第3正規形: 推移関数従属の除去

## 試験での頻出パターン
- 正規化のメリット・デメリット
- 正規化レベルの判定問題
```
<br><br>
### ◉毎朝の過去問共有
- **方法**: メッセージでスクショ送付 + 通話orメッセージで解説
- **メリット**: 素早い共有、リアルタイム議論
- **Issue化**: 特に重要な問題は後から重点問題としてIssue作成
<br><br>
#### 学んだことノート例
```
タイトル: [データベース] 正規化の完全解説
ラベル: Knowledge

内容:
## 担当者
@rin5uron

## 解説内容
### 第1正規形（1NF）
- 繰り返し項目の排除
- 例: 住所1,住所2 → 住所テーブル分離

### 実際の問題での出題パターン
- 非正規化→正規化への変換問題
- メリット・デメリットの選択問題
```
