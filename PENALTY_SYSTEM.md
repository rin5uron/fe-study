# 💰 ペナルティシステム

## 📋 概要

毎日の勉強記録を提出しない場合、**3000円のペナルティ**が発生します。
このシステムは、GitHub Actionsで自動的にチェックされ、未提出日を記録します。

## 🎯 ルール

1. **毎日、勉強記録のIssueを作成する**
   - タイトルに日付を含める（例: `[勉強記録] 2026-01-21`）
   - 自分のユーザー名ラベルを付ける（例: `user-rin5uron`）

2. **チェック時間**
   - 毎日朝5:00 JSTに自動チェック（前日の記録をチェック）
   - 前日の記録がない場合、ペナルティが記録されます

3. **ペナルティ**
   - 未提出日: **3000円**
   - ペナルティは「💰 ペナルティ記録」Issueに自動記録されます

## 📝 使い方

### 1. 勉強記録Issueの作成

1. 「New Issue」をクリック
2. 「毎日の勉強記録」テンプレートを選択
3. 日付と学習内容を記入
4. **自分のユーザー名ラベルを追加**（重要！）
   - 例: `user-rin5uron`, `user-partner` など
5. Issueを作成

### 2. ユーザー名ラベルの設定

#### 新しい参加者を追加する場合

1. リポジトリの「Labels」ページに移動
2. 「New label」をクリック
3. ラベル名: `user-{ユーザー名}`（例: `user-rin5uron`）
4. 色を設定（例: 青系）
5. ラベルを作成

#### GitHub Actionsの設定を更新

`.github/workflows/check-daily-study.yml` の以下の部分を編集：

```yaml
const users = ['user-rin5uron', 'user-partner']; // 参加者のラベルを追加
```

### 3. ペナルティの確認

- 「💰 ペナルティ記録」Issueで確認できます
- 毎日のチェック結果がコメントとして追加されます

## 🔧 手動チェック

GitHub Actionsのページから「Run workflow」をクリックして、手動でチェックを実行することもできます。

## ⚠️ 注意事項

- Issueのタイトルに日付を含めるか、本文に日付を記載してください
- 必ず自分のユーザー名ラベルを付けてください（チェックのため）
- その日のうちに記録を作成してください（翌朝5時のチェックで検出されます）

## 📊 ペナルティの集計

ペナルティ記録Issueにコメントとして記録されるため、手動で集計するか、後で自動集計機能を追加する予定です。
